2. Database Schema Details

Table Users

Column Data Type Constraints Description
user_id INT PK, Auto Increment Unique identifier for the user.
username VARCHAR(50) UNIQUE, NOT NULL Username for system login.
password_hash VARCHAR(255) NOT NULL Hashed password to ensure security.
role ENUM 'Admin', 'Support' Defines user privileges and access levels.

Table Nodes

Column Data Type Constraints Description
node_id INT PK, Auto Increment Unique identifier for the network node.
ip_address VARCHAR(45) UNIQUE, NOT NULL IP address of the server or BTS element.
name VARCHAR(100) NOT NULL Alias for the node (e.g., BTS_Dandy_Mall).
status ENUM 'OK', 'Issue' Current operational status of the node (Green/Red).

Table Trap_History

Column Data Type Constraints Description
trap_id INT PK, Auto Increment Unique identifier for the received SNMP trap.
node_id INT FK (Nodes.node_id) The node that generated and sent the trap.
error_description TEXT NOT NULL Details of the error (e.g., CPU is 100%, Storage Full).
received_at DATETIME DEFAULT CURRENT_TIMESTAMP Timestamp marking when the trap was captured.
is_resolved BOOLEAN DEFAULT FALSE Status flag indicating if the issue was manually fixed.
resolved_by INT FK (Users.user_id), NULL Tracks which user marked the issue as resolved.

Table Trap_Actions

Column Data Type Constraints Description
action_id INT PK, Auto Increment Unique identifier for the configured action.
node_id INT FK (Nodes.node_id) The node associated with this automated action.
action_type ENUM 'SMS', 'Email', 'Script' The type of automated response to execute.
target_value VARCHAR(255) NOT NULL The target destination (Phone number, Email, or Script path).
